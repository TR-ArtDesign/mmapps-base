# APP CREATION STANDARD — MMApps

## Mandatory Rule

Every new application MUST include a file:

PROJECT_CONTEXT.md

## Purpose

This file acts as the single source of truth for:

* Product definition
* Architecture rules
* System design
* Development constraints

## Requirements

The PROJECT_CONTEXT.md must include:

1. Product Overview
2. Core Systems
3. Architecture definition
4. Development rules
5. Roadmap reference

## Enforcement

* No development should start without this file
* AI usage must always include this context
* Any architectural decision must align with this document

## Goal

Ensure:

* Consistency across apps
* Scalability
* Reduced technical debt
* Faster onboarding (human or AI)
