# SKILL.md — M.M.APPS MASSIVE RETENTION CASUAL APPS ENGINE (V1.0)

## 🎯 OBJECTIVE

Create fully functional, store-ready mobile apps focused on:

* High retention
* Offline-first gameplay (when applicable)
* Scalable architecture
* Fast production
* Monetization via ads (optimized, non-intrusive)

The system MUST generate apps that are:

* Immediately compliant with Google Play and Apple App Store guidelines
* Designed for user retention loops
* Lightweight and performant
* Ready for publication without rework

---

## 🧱 CORE PRINCIPLES

### 1. RETENTION FIRST (NON-NEGOTIABLE)

Every app MUST include:

* Daily return incentive (streak or reward)
* Progress system (levels, ranking, or unlockables)
* Feedback loop (visual, sound, reward)
* Session continuity (resume or evolving state)

---

### 2. OFFLINE-FIRST ARCHITECTURE

* Core functionality MUST work without internet
* Ads and optional features MAY depend on connection
* Cache critical assets locally

---

### 3. LOW FRICTION UX

* Instant usability (no onboarding overload)
* Maximum 1–2 taps to start main action
* Clean, intuitive interface

---

### 4. PERFORMANCE & SIZE

* App must load in <2 seconds
* Minimal memory usage
* Optimized assets
* Keep APK/IPA size as small as possible

---

## 📱 TECH STACK (DEFAULT)

* Framework: React Native (preferred) or Flutter
* State management: Zustand / Redux Toolkit
* Storage: AsyncStorage / SQLite
* Ads: AdMob
* Notifications: Firebase Cloud Messaging
* Analytics: Firebase Analytics

---

## 🧩 APP STRUCTURE (MANDATORY)

### 1. HOME SCREEN

* Primary action visible immediately
* Resume last session (if applicable)
* Access to rewards / daily system

---

### 2. CORE FEATURE (GAME OR CONTENT)

* Must be interactive within 3 seconds
* Must include progression or variation
* Must not feel static

---

### 3. PROGRESSION SYSTEM

At least one must exist:

* Levels
* Score ranking
* Unlockable themes/modes
* Time-based rewards

---

### 4. DAILY SYSTEM (MANDATORY)

* Daily reward
* Streak counter
* Return incentive

---

### 5. SETTINGS

* Sound toggle
* Notifications toggle
* Privacy Policy
* Terms of Use

---

## 🔁 RETENTION ENGINE (MANDATORY)

Every app MUST include:

### 1. STREAK SYSTEM

* Daily login tracking
* Visual reinforcement

---

### 2. REWARD LOOP

* Coins / points / unlockables
* Visible progression

---

### 3. VARIABLE REWARD

* Random bonus triggers
* Surprise rewards

---

### 4. SESSION HOOK

* “One more action” trigger
* Examples:

  * “Next reward in 1 play”
  * “Almost there”

---

## 💰 MONETIZATION RULES

### AD TYPES

* Interstitial Ads (controlled frequency)
* Rewarded Ads (priority)
* Banner Ads (optional, non-intrusive)

---

### RULES

* NEVER show ads on first interaction
* NEVER interrupt core gameplay abruptly
* Rewarded ads MUST be optional and valuable

---

### FREQUENCY CONTROL

* Interstitial every X sessions/actions
* Cooldown between ads

---

## 🧠 ENGAGEMENT MECHANICS

Each app MUST include at least TWO:

* Time pressure
* Score challenge
* Personalization
* Unlockable content
* Visual evolution
* Competitive self-progression

---

## 🎨 UI/UX GUIDELINES

* Follow Material Design / Apple HIG
* Strong visual feedback is mandatory

### REQUIRED MICROINTERACTIONS

* Button press feedback
* Success animation
* Reward feedback

---

### DESIGN SYSTEM

* Primary color
* Secondary color
* Feedback colors (success, error)
* Typography hierarchy

---

## 🔔 NOTIFICATIONS (RECOMMENDED)

* Daily reminder
* Reward availability
* Streak warning

---

## 🔒 STORE COMPLIANCE (CRITICAL)

### MUST INCLUDE

* Privacy Policy URL
* Terms of Use
* Data transparency
* Consent for ads (GDPR / LGPD)

---

### MUST AVOID

* App crashes
* Misleading UI
* Fake engagement
* Excessive ads

---

## 📦 DELIVERY FORMAT (WHEN GENERATING APPS)

The system MUST output:

1. App concept
2. Feature breakdown
3. Screen structure
4. UX behavior
5. Retention system
6. Monetization strategy
7. Technical stack
8. Store compliance checklist

---

## ⚠️ HARD RESTRICTIONS

* DO NOT create static apps
* DO NOT rely only on UI as differentiation
* DO NOT overload with ads
* DO NOT skip retention systems

---

## 🚀 PERFORMANCE REQUIREMENTS

* First interaction <3 seconds
* No UI blocking
* Smooth transitions

---

## 🧪 QUALITY REQUIREMENTS

* No crashes
* Predictable behavior
* Clean state management

---

## 🔄 SCALABILITY

Apps MUST support:

* New game modes
* Theming
* Feature toggles

---

## ✅ SUCCESS CRITERIA

An app is valid ONLY if:

* It is usable within 3 seconds
* It has a retention loop
* It has monetization correctly implemented
* It meets store approval guidelines
* It encourages repeated sessions

---

## 📌 FINAL PRINCIPLE

This system does NOT create apps.

It creates:

→ Retention-driven interactive products designed for scale and monetization.
